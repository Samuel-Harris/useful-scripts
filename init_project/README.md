# init_project - Interactive Project Scaffolding Tool

## DESCRIPTION

A CLI tool that interactively scaffolds new projects with opinionated defaults for Python development. It automates the setup of common development tools like `uv`, `ruff`, `pyright`, `pre-commit`, and optionally `pytest`, along with GitHub CI workflows and AI assistant ignore files.

## USAGE

```bash
cd init_project
uv run main.py [project_directory]
```

- `project_directory` - Path where the project will be initialized (default: current directory)

## INTERACTIVE PROMPTS

The script will guide you through the following configuration options:

1. **Create directory?** - If the target directory doesn't exist, you'll be asked whether to create it
2. **Python project?** - Whether to set up a Python project with `uv`
3. **GitHub CI workflow?** - Whether to add a `.github/workflows/ci.yml` file
4. **Python version** - Which Python version to use (default: 3.14)
5. **Type checking mode** - Pyright strictness level: `off`, `basic`, `standard`, or `strict`
6. **pytest?** - Whether to include pytest and create a test directory
7. **.cursorignore?** - Whether to add a `.cursorignore` file for Cursor AI
8. **.geminiignore?** - Whether to add a `.geminiignore` file for Gemini AI

## GENERATED PROJECT STRUCTURE

For a Python project with all options enabled:

```
project_root/
├── .github/
│   └── workflows/
│       └── ci.yml              # GitHub Actions CI workflow
├── src/
│   ├── app/
│   │   ├── __init__.py
│   │   └── main.py             # Hello world starter file
│   └── test/
│       ├── __init__.py
│       └── test_main.py        # Starter test file
├── .cursorignore               # Cursor AI ignore patterns
├── .geminiignore               # Gemini AI ignore patterns
├── .pre-commit-config.yaml     # Pre-commit hooks configuration
├── .python-version             # Python version file
├── pyproject.toml              # Project configuration (generated by uv)
├── pyrightconfig.json          # Pyright type checking configuration
└── uv.lock                     # Dependency lock file
```

## TOOLS CONFIGURED

| Tool           | Purpose                                    |
| -------------- | ------------------------------------------ |
| **uv**         | Fast Python package installer and resolver |
| **ruff**       | Fast Python linter and formatter           |
| **pyright**    | Static type checker for Python             |
| **pre-commit** | Git hooks framework for code quality       |
| **pytest**     | Testing framework (optional)               |

## EXAMPLE

```bash
# Initialize a new project in a new directory
uv run main.py ~/projects/my-new-app

# Initialize in the current directory
uv run main.py .
```

## DEPENDENCIES

- Python 3.14+
- [uv](https://github.com/astral-sh/uv) - Must be installed globally
- jinja2, pydantic, rich (installed automatically via `uv sync`)

## NOTES

- The script will skip creating files that already exist, making it safe to run on partially initialized projects
- All development dependencies (ruff, pyright, pre-commit) are mandatory for Python projects
- The CI workflow automatically runs linting, type checking, and tests on push/PR
